15class AsyncServer(base_server.BaseServer):
16 """An Engine.IO server for asyncio.
17
18 This class implements a fully compliant Engine.IO web server with support
19 for websocket and long-polling transports, compatible with the asyncio
20 framework on Python 3.5 or newer.
21
22 :param async_mode: The asynchronous model to use. See the Deployment
23 section in the documentation for a description of the
24 available options. Valid async modes are "aiohttp",
25 "sanic", "tornado" and "asgi". If this argument is not
26 given, "aiohttp" is tried first, followed by "sanic",
27 "tornado", and finally "asgi". The first async mode that
28 has all its dependencies installed is the one that is
29 chosen.
30 :param ping_interval: The interval in seconds at which the server pings
31 the client. The default is 25 seconds. For advanced
32 control, a two element tuple can be given, where
33 the first number is the ping interval and the second
34 is a grace period added by the server.
35 :param ping_timeout: The time in seconds that the client waits for the
36 server to respond before disconnecting. The default
37 is 20 seconds.
38 :param max_http_buffer_size: The maximum size that is accepted for incoming
39 messages. The default is 1,000,000 bytes. In
40 spite of its name, the value set in this
41 argument is enforced for HTTP long-polling and
42 WebSocket connections.
43 :param allow_upgrades: Whether to allow transport upgrades or not.
44 :param http_compression: Whether to compress packages when using the
45 polling transport.
46 :param compression_threshold: Only compress messages when their byte size
47 is greater than this value.
48 :param cookie: If set to a string, it is the name of the HTTP cookie the
49 server sends back tot he client containing the client
50 session id. If set to a dictionary, the ``'name'`` key
51 contains the cookie name and other keys define cookie
52 attributes, where the value of each attribute can be a
53 string, a callable with no arguments, or a boolean. If set
54 to ``None`` (the default), a cookie is not sent to the
55 client.
56 :param cors_allowed_origins: Origin or list of origins that are allowed to
57 connect to this server. Only the same origin
58 is allowed by default. Set this argument to
59 ``'*'`` to allow all origins, or to ``[]`` to
60 disable CORS handling.
61 :param cors_credentials: Whether credentials (cookies, authentication) are
62 allowed in requests to this server.
63 :param logger: To enable logging set to ``True`` or pass a logger object to
64 use. To disable logging set to ``False``. Note that fatal
65 errors are logged even when ``logger`` is ``False``.
66 :param json: An alternative json module to use for encoding and decoding
67 packets. Custom json modules must have ``dumps`` and ``loads``
68 functions that are compatible with the standard library
69 versions.
70 :param async_handlers: If set to ``True``, run message event handlers in
71 non-blocking threads. To run handlers synchronously,
72 set to ``False``. The default is ``True``.
73 :param monitor_clients: If set to ``True``, a background task will ensure
74 inactive clients are closed. Set to ``False`` to
75 disable the monitoring task (not recommended). The
76 default is ``True``.
77 :param transports: The list of allowed transports. Valid transports
78 are ``'polling'`` and ``'websocket'``. Defaults to
79 ``['polling', 'websocket']``.
80 :param kwargs: Reserved for future extensions, any additional parameters
81 given as keyword arguments will be silently ignored.
82 """
