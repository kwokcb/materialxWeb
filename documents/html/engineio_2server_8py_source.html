13 """An Engine.IO server.
14
15 This class implements a fully compliant Engine.IO web server with support
16 for websocket and long-polling transports.
17
18 :param async_mode: The asynchronous model to use. See the Deployment
19 section in the documentation for a description of the
20 available options. Valid async modes are "threading",
21 "eventlet", "gevent" and "gevent_uwsgi". If this
22 argument is not given, "eventlet" is tried first, then
23 "gevent_uwsgi", then "gevent", and finally "threading".
24 The first async mode that has all its dependencies
25 installed is the one that is chosen.
26 :param ping_interval: The interval in seconds at which the server pings
27 the client. The default is 25 seconds. For advanced
28 control, a two element tuple can be given, where
29 the first number is the ping interval and the second
30 is a grace period added by the server.
31 :param ping_timeout: The time in seconds that the client waits for the
32 server to respond before disconnecting. The default
33 is 20 seconds.
34 :param max_http_buffer_size: The maximum size that is accepted for incoming
35 messages. The default is 1,000,000 bytes. In
36 spite of its name, the value set in this
37 argument is enforced for HTTP long-polling and
38 WebSocket connections.
39 :param allow_upgrades: Whether to allow transport upgrades or not. The
40 default is ``True``.
41 :param http_compression: Whether to compress packages when using the
42 polling transport. The default is ``True``.
43 :param compression_threshold: Only compress messages when their byte size
44 is greater than this value. The default is
45 1024 bytes.
46 :param cookie: If set to a string, it is the name of the HTTP cookie the
47 server sends back tot he client containing the client
48 session id. If set to a dictionary, the ``'name'`` key
49 contains the cookie name and other keys define cookie
50 attributes, where the value of each attribute can be a
51 string, a callable with no arguments, or a boolean. If set
52 to ``None`` (the default), a cookie is not sent to the
53 client.
54 :param cors_allowed_origins: Origin or list of origins that are allowed to
55 connect to this server. Only the same origin
56 is allowed by default. Set this argument to
57 ``'*'`` to allow all origins, or to ``[]`` to
58 disable CORS handling.
59 :param cors_credentials: Whether credentials (cookies, authentication) are
60 allowed in requests to this server. The default
61 is ``True``.
62 :param logger: To enable logging set to ``True`` or pass a logger object to
63 use. To disable logging set to ``False``. The default is
64 ``False``. Note that fatal errors are logged even when
65 ``logger`` is ``False``.
66 :param json: An alternative json module to use for encoding and decoding
67 packets. Custom json modules must have ``dumps`` and ``loads``
68 functions that are compatible with the standard library
69 versions.
70 :param async_handlers: If set to ``True``, run message event handlers in
71 non-blocking threads. To run handlers synchronously,
72 set to ``False``. The default is ``True``.
73 :param monitor_clients: If set to ``True``, a background task will ensure
74 inactive clients are closed. Set to ``False`` to
75 disable the monitoring task (not recommended). The
76 default is ``True``.
77 :param transports: The list of allowed transports. Valid transports
78 are ``'polling'`` and ``'websocket'``. Defaults to
79 ``['polling', 'websocket']``.
80 :param kwargs: Reserved for future extensions, any additional parameters
81 given as keyword arguments will be silently ignored.
